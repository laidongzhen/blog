# QMS 质量管理系统项目技术文档

## 项目背景（Situation）

随着制造业数字化转型的深入，企业对质量管理系统（QMS）的需求日益增长。为了提升产品质量管控能力，实现质量数据的实时采集、分析和追溯，公司启动了 QMS 3.0 项目，旨在构建一个功能完善、用户友好、技术先进的质量管理平台。

项目面临的挑战包括：
- 业务逻辑复杂，涉及多个部门的协作
- 数据量大，需要高效的表格处理和数据可视化能力
- 系统集成需求高，需要与其他系统无缝对接
- 用户体验要求高，需要提供直观、易用的界面

## 核心功能模块（Tasks）

### 1. 组件库（src/components）

#### 表格组件系列
- **CommonTableNew**：新一代表格组件，支持列配置、方案保存、拖拽排序等功能
- **AgGridTable**：基于 AG Grid 的高性能表格组件，支持复杂数据处理
- **CommonTablePlus**：增强型表格组件，支持多方案管理和高级搜索

#### 表单组件系列
- **MaterialSelectComplex**：复合下拉组件，支持触底加载、远程搜索
- **CommonSelectDialog**：通用选择对话框，支持多条件搜索和批量选择
- **FileUpload**：文件上传组件，支持多文件上传和进度显示

#### 布局和通用组件
- **Layout**：系统布局组件，支持响应式设计
- **Dialog**：自定义对话框组件，支持多种配置
- **Breadcrumb**：面包屑导航组件，提升用户体验

### 2. 插件（src/plugins）

- **MaterialSelectComplex**：物料选择插件，支持复杂的物料筛选和加载
- **validateDialog**：验证对话框插件，用于表单验证和提示

### 3. API 接口（src/api）

- **quality-manage.js**：质量管理相关 API
- **common.js**：通用 API 接口
- **user.js**：用户管理相关 API
- **product-management.js**：产品管理相关 API

## 技术亮点与解决方案（Actions）

### 1. 高性能表格解决方案
- **挑战**：系统需要处理大量质量数据，传统表格组件性能不足
- **解决方案**：
  - 集成 AG Grid 实现高性能数据渲染和处理
  - 开发自定义表格组件，支持虚拟滚动和按需加载
  - 实现列配置方案保存，提升用户体验

### 2. 复杂表单控件
- **挑战**：业务场景需要复杂的下拉选择功能，如物料选择、标准选择等
- **解决方案**：
  - 开发复合下拉组件，支持触底加载、远程搜索
  - 实现自定义指令 `loadMore`，支持滚动触底加载更多数据
  - 提供灵活的配置选项，适配不同业务场景

### 3. 微前端架构
- **挑战**：系统功能模块多，需要独立开发和部署
- **解决方案**：
  - 采用 Qiankun 实现微前端架构
  - 将系统拆分为多个独立子应用，提升开发效率
  - 实现子应用间的通信和状态共享

### 4. 数据可视化
- **挑战**：需要直观展示质量数据趋势和分析结果
- **解决方案**：
  - 集成 ECharts 4 和 ECharts 5，提供丰富的图表类型
  - 开发自定义图表组件，满足特定业务需求
  - 实现图表数据的实时更新和交互
